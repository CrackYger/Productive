# Architecture Audits

## Zweck
Datierte Snapshots, Phasen-Audits und historische Umsetzungsnotizen für `Productive`.

## Lese-Regel
- Audits sind historisches Gedächtnis, keine aktiven Regeln.
- Wenn ein Audit veraltet ist: nicht löschen, sondern als überholt markieren.

## Dateien in diesem Ordner

*(leer — Projekt in Prototyp-Phase. Erster Audit nach Phase 1 geplant.)*

## Wann eine neue Datei anlegen?
- Nach Abschluss einer Phase: Snapshot des erreichten Stands
- Bei einer größeren Refaktorierung: Vorher-/Nachher-Dokumentation
- Leitfrage: „In zwölf Monaten noch aktiv steuernd?" → nein, aber historisch nützlich → Audit

## Namensschema
- `phase_<N>_<name>_audit.md`
- `<thema>_<YYYY-MM-DD>.md`
