# Architecture Rules

## Zweck
Dauerhafte technische und fachliche Architekturregeln für `Productive`.

## Lese-Regel
- Vor einem Eingriff in einen Bereich die zuständige Datei hier lesen.
- Wenn noch keine Datei existiert: Entscheidung treffen, dokumentieren, hier ablegen.

## Dateien in diesem Ordner

*(leer — Projekt in Prototyp-Phase. Erste Architektur-Regeln entstehen in Phase 1.)*

## Wann eine neue Datei anlegen?
- Wenn eine dauerhafte Architekturentscheidung für einen Bereich getroffen wurde
- Leitfrage: „In zwölf Monaten noch aktiv steuernd?" → ja → Datei anlegen

## Namensschema
- `<bereich>_rules.md` (z.B. `task_management_rules.md`, `auth_rules.md`)
- Alternativ bei phasengebundenem Inhalt: `phase_<N>_<bereich>.md`
